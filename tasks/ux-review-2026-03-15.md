# UX Expert Review — UcetniWebApp
**Datum:** 2026-03-15
**Typ:** Kompletní UX audit celé aplikace

---

## Shrnutí

Identifikováno **40 nálezů** ve 3 portálech + landing page. Celková úroveň UX je solidní (7/10), ale existují systémové problémy v přístupnosti, error handlingu a konzistenci nových modulů.

---

## P0 — OKAMŽITĚ (brání konverzím / poškozuje brand)

### UX-01: Hero překlep "Účetnictvi" → "Účetnictví"
- **Soubor:** `components/landing/hero.tsx`
- **Dopad:** První věc co návštěvník vidí. Překlep v hlavním titulku snižuje důvěryhodnost.

### UX-02: CTA "Začít zdarma" odkazuje na login místo registrace
- **Soubor:** `components/landing/hero.tsx` + `components/landing/cta-section.tsx`
- **Problém:** Odkaz vede na `/auth/login` místo `/auth/register`. Nový návštěvník je zmaten přihlašovacím formulářem.
- **Fix:** Změnit href na `/auth/register`.

### UX-03: Registrační stránka — chybí layout wrapper
- **Soubor:** `app/auth/register/page.tsx`
- **Problém:** Holá `<Card>` bez pozadí, centering, loga — zatímco login má gradient, blur efekty, branding. Narušuje důvěru při registraci.
- **Fix:** Přidat wrapper s gradient pozadím a logem (shodně s login page).

## P1 — VYSOKÁ PRIORITA (UX / přístupnost / funkčnost)

### UX-04: Footer — právní odkazy nefungují
- **Soubor:** `components/landing/footer.tsx`
- **Problém:** "Obchodní podmínky", "Ochrana osobních údajů", "GDPR" jsou `<span>`, ne `<Link>`. Neklikatelné.
- **Fix:** Vytvořit placeholder stránky nebo zobrazit jako modal.

### UX-05: Pricing billing toggle bez ARIA
- **Soubor:** `components/landing/pricing-section.tsx`
- **Problém:** Přepínač měsíční/roční nemá `role="switch"`, `aria-checked`, `aria-label`. Screen reader nerozpozná stav.

### UX-06: Dynamický Tailwind grid nefunguje v produkci
- **Soubor:** `components/pricing-table.tsx`
- **Problém:** `` grid-cols-${plans.length} `` — Tailwind JIT/purge nedetekuje dynamické class names. V produkci grid nebude mít správný počet sloupců.
- **Fix:** Podmíněné třídy: `plans.length === 3 ? 'md:grid-cols-3' : 'md:grid-cols-2'`

### UX-07: Klientská mobilní navigace — chybějící sekce
- **Soubor:** `app/client/layout.tsx`
- **Problém:** Mobilní bottom tab bar zobrazuje jen 5 z 10 sekcí. Cesťák, Dotazník, Dohodáři, Spisy, Předplatné jsou na mobilu **nedostupné** (nejsou v tab baru ani v hamburger menu).
- **Fix:** Přidat "Více" menu na mobilní lištu (jako má účetní portál).

### UX-08: Tiché error handling na 15+ stránkách
- **Pattern:** `catch(() => {})` nebo `catch(() => setData([]))` bez uživatelské zpětné vazby.
- **Postižené stránky:**
  - Accountant: clients list, analytics, admin/people, extraction/clients, knowledge-base CRUD, komunikace, deadlines, dashboard
  - Client: dashboard (3× fetch), messages, travel, dohodari (vč. submit výkazu!), tax-questionnaire
- **Dopad:** Uživatel vidí prázdný stav místo chybové hlášky. Neví jestli nemá data nebo API selhalo.
- **Fix:** Vytvořit `<ErrorState>` komponentu a `useErrorToast()` hook. Použít konzistentně.

### UX-09: Knowledge-base — `confirm()` místo AlertDialog
- **Soubor:** `app/accountant/knowledge-base/page.tsx`
- **Problém:** Nativní `confirm()` dialog při mazání článku. Na mobilech vypadá špatně, nerespektuje design systém.
- **Fix:** Nahradit `<AlertDialog>` z Radix UI.

### UX-10: Fixed bottom bar bez safe-area padding
- **Soubory:** `app/client/tax-questionnaire/page.tsx`, `app/client/onboarding/page.tsx`
- **Problém:** Fixed bottom bar na iOS Safari přesahuje bezpečnou zónu (překryt home indikátorem).
- **Fix:** Přidat `pb-[env(safe-area-inset-bottom)]` nebo `pb-safe`.

## P2 — STŘEDNÍ PRIORITA (konzistence / kvalita)

### UX-11: DVĚ různé pricing komponenty s duplicitními daty
- **Soubory:** `components/landing/pricing-section.tsx` vs `components/pricing-table.tsx`
- **Problém:** Dva nezávislé ceníkové komponenty s mírně odlišnými popisy a daty. Riziko rozchodu.
- **Fix:** Centralizovat do `lib/pricing-plans.ts`, importovat v obou.

### UX-12: Nativní `<select>` místo design systému (5 míst)
- **Soubory:** dohodari/page.tsx, agreements/page.tsx, tax-questionnaire/page.tsx, onboarding/page.tsx, knowledge-base/page.tsx
- **Problém:** Nativní HTML `<select>` nerespektuje vizuální styl aplikace (Radix Select).

### UX-13: `window.location.href` místo `router.push()` v plan gates
- **Soubory:** dohodari/page.tsx, agreements/page.tsx
- **Problém:** Full page reload místo client-side navigace. Pomalé, ztrácí stav.

### UX-14: Chybí `aria-expanded` na expandovatelných řádcích
- **Soubory:** dohodari/page.tsx (klient + účetní), knowledge-base/page.tsx
- **Problém:** Klikatelné `<div>` bez `role="button"`, `aria-expanded`, `tabIndex`.

### UX-15: Icon-only buttony bez `aria-label`
- **Soubory:** knowledge-base (edit/delete), agreements (PDF download), extraction (camera)
- **Problém:** Mají `title` atribut ale ne `aria-label`. Screen readery `title` nemusí přečíst.

### UX-16: Registrace — chybí validace hesla
- **Soubor:** `app/auth/register/page.tsx`
- **Problém:** Jen `minLength={6}` HTML atribut. Chybí vizuální feedback o síle hesla, kontrola shody hesel před odesláním.

### UX-17: Subscription stránka — chybí loading state
- **Soubor:** `app/client/subscription/page.tsx`
- **Problém:** `usePlanFeatures()` vrací `loading`, ale stránka ho ignoruje. Blikání při načítání (zobrazí Free → pak skutečný plán).

### UX-18: Dva různé spinner patterny
- **Accountant:** CSS spinner (`animate-spin border-b-2 border-purple-600`)
- **Client:** Mix Lucide `Loader2` a CSS spinneru
- **Fix:** Zvolit jeden a standardizovat.

### UX-19: Badge color nekonzistence v accountant layout
- **Soubor:** `app/accountant/layout.tsx`
- **Problém:** Desktop "Práce" badge `bg-violet-400`, mobilní `bg-amber-400` — dva různé barvy pro stejný element.

### UX-20: Analytics texty bez diakritiky
- **Soubor:** `app/accountant/analytics/page.tsx`
- **Problém:** "Rust firmy", "Mesicni prijem", "Cisty rust letos" — nedokončený překlad.

### UX-21: Submit disabled bez feedbacku
- **Soubory:** tax-questionnaire (< 50%), onboarding (< 30%)
- **Problém:** Tlačítko "Odeslat" je disabled bez vysvětlení. Uživatel neví kolik musí vyplnit.
- **Fix:** Tooltip nebo text "Vyplňte alespoň 50% otázek".

## P3 — NÍZKÁ PRIORITA (polish / nice-to-have)

- **Soubor:** `app/auth/login/page.tsx`

### UX-23: Navbar — chybí aktivní stav při scrollování landing page
- **Soubor:** `components/landing/navbar.tsx`

### UX-24: Invoices header přetéká na mobilu
- **Soubor:** `app/accountant/invoices/page.tsx`
- **Problém:** H1 + 3 tlačítka v jednom `flex` řádku bez `flex-wrap`.

### UX-25: Requests tabulka bez overflow-x-auto
- **Soubor:** `app/accountant/requests/page.tsx`
- **Problém:** `<Table>` přetéká na mobilech.

### UX-26: Analytics graf — žádné ARIA atributy
- **Soubor:** `app/accountant/analytics/page.tsx`
- **Problém:** Custom bar chart bez `role="img"`, `aria-label`. Tooltip jen na hover (ne klávesnicí).

### UX-27: Komunikace — hardcoded výška
- **Soubor:** `app/accountant/komunikace/page.tsx`
- **Problém:** `style={{ height: 'calc(100vh - 160px)' }}` — na mobilech s dynamic browser chrome může být špatná.

### UX-28: Knowledge-base — formuláře bez `<label>`
- **Soubor:** `app/accountant/knowledge-base/page.tsx`
- **Problém:** `<input>` a `<textarea>` s jen `placeholder`, bez `<label htmlFor>`.

### UX-29: Extraction filtry skryté na mobilu bez náhrady
- **Soubor:** `app/accountant/extraction/clients/page.tsx`
- **Problém:** `hidden sm:flex` — na mobilu nejsou filtry vůbec dostupné.

### UX-30: Dohodáři — dva moduly se překrývajícím obsahem
- **Soubory:** `clients/[companyId]/dohodari/` vs `clients/[companyId]/agreements/`
- **Problém:** Oba zobrazují dohodáře z různých zdrojů. Uživatel může vidět nekonzistentní data.

### UX-31: Nekonzistentní threshold pro submit dotazníků
- **Soubory:** tax-questionnaire (50%) vs onboarding (30%)

### UX-32: Travel stats tab — chybí empty state
- **Soubor:** `app/client/travel/page.tsx`
- **Problém:** Když `stats === null`, zobrazí se prázdný `<div>`.

### UX-33: Cases badge barvy bez dark mode variant
- **Soubor:** `app/client/cases/page.tsx`

### UX-34: Dohodari hardcoded `text-gray-300` místo `text-muted-foreground`
- **Soubor:** `app/client/dohodari/page.tsx`

### UX-35: Knowledge-base — chybí plan gate
- **Soubor:** `app/accountant/knowledge-base/page.tsx`
- **Problém:** V pricing table je KB jako Business-only feature, ale stránka nemá feature gate check.

## ARIA / Accessibility — celkové hodnocení

| Portál | ARIA stav | Poznámka |
|--------|-----------|----------|
| Landing page | ČÁSTEČNÉ | Navbar má aria-label + aria-expanded, zbytek slabý |
| Login | DOBRÉ | aria-label na password toggle, label htmlFor |
| Register | CHYBÍ | Žádné ARIA |
| Accountant portál | SLABÉ | 8 ARIA výskytů ve 3 souborech z 25+. Dashboard nejlepší (role, tabIndex, aria-describedby) |
| Client portál | CHYBÍ | 0 ARIA atributů v celém adresáři /app/client/ |

**Systémový problém:** Klientský portál nemá **žádnou** přístupnostní podporu. Účetní portál má zárodky (dashboard), ale většina stránek je bez ARIA.

## Doporučené pořadí oprav

1. **P0 (3 nálezy)** — překlep, CTA link, register layout (~1 hodina)
2. **Error handling systém** (UX-08) — `<ErrorState>` + `useErrorToast()` (~2 hodiny, dotkne se 15+ souborů)
3. **Mobilní navigace klienta** (UX-07) — "Více" menu (~1 hodina)
4. **Pricing konsolidace** (UX-06 + UX-11) — Tailwind fix + centrální data (~1 hodina)
5. **ARIA sweep** — `aria-label`, `aria-expanded`, `role` na nové moduly (~2 hodiny)
6. **Nativní select → Radix Select** (UX-12) — 5 souborů (~1 hodina)
7. **P3 polish** — postupně podle priorit
